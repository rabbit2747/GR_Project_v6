## 2. 4개 제품 라인

### 2.1 GR 자동 진단 (Automated Security Diagnosis)

**핵심 가치**: 인프라 구성을 입력하면 GR 좌표 기반으로 보안 취약점과 정책 미준수를 자동 발견

**작동 원리**:
1. 고객 인프라 구성 스캔 또는 수동 입력
2. 각 구성요소에 GR 좌표 자동 할당
3. 좌표 기반 정책 상속 → 현재 상태와 비교
4. 미준수 항목, 누락된 통제, 취약점 리포트 생성
5. GR Atlas로 시각화하여 우선순위 제시

**차별화**: 기존 진단 도구는 개별 취약점만 찾지만, GR 진단은 **인프라 맥락(좌표) 기반**으로 위험도를 재산정. Zone 3의 취약점 ≠ Zone 1의 같은 취약점.

### 2.2 GR IaC (Infrastructure as Code)

**핵심 가치**: GR 좌표 기반 보안 정책이 내장된 IaC 템플릿 자동 생성

**작동 원리**:
1. 원하는 인프라 구성을 좌표로 정의 (예: "L5, Z3에 PostgreSQL")
2. 좌표 기반 보안 정책 자동 결정
3. Terraform/Ansible 코드 자동 생성 (보안 정책 포함)
4. 배포 전 GR 진단으로 사전 검증
5. 배포 후 GR Atlas로 실시간 모니터링

**차별화**: 기존 IaC는 인프라만 배포하고 보안은 별도. GR IaC는 **좌표 기반 보안 정책이 코드에 내장**.

**비전**: CVE 발표 → 영향받는 좌표 자동 식별 → 취약 환경을 IaC로 자동 재현 → 패치 검증 → 안전한 IaC 배포. 이것이 **CVE → IaC Lab 자동 생성** 비전.

### 2.3 GR Education

**핵심 가치**: 실제 인프라 맥락에서 보안을 배우는 인터랙티브 교육 플랫폼

**작동 원리**:
1. GR Atlas로 실제 인프라 아키텍처 시각화
2. 공격 경로를 좌표 위에서 시뮬레이션
3. 학습자가 방어 조치를 좌표에 배치
4. 즉시 피드백: 정책 준수율, 방어 커버리지 변화

**교육 모듈 예시**:
- "Zone 0A → Zone 3 공격 경로 분석"
- "Layer 5 데이터 서비스 보안 정책 설계"
- "CVE-2024-XXXX가 당신의 아키텍처에 미치는 영향"

### 2.4 GR Consulting

**핵심 가치**: GR 좌표 체계를 활용한 체계적 보안 컨설팅

**제공 서비스**:
- 인프라 보안 아키텍처 리뷰 (GR 좌표 매핑)
- 보안 정책 갭 분석 (좌표 기반 자동 진단)
- 컴플라이언스 매핑 (NIST, ISO 27001 → GR 좌표)
- 인시던트 대응 계획 수립 (GR Atlas 기반 블래스트 반경 분석)

## 3. 제품 시너지 사이클

```
     ┌──── GR 진단 ────┐
     │  (문제 발견)      │
     ↓                  │
GR Consulting          │
  (전문 분석)     ←──── ↑
     ↓                  │
  GR IaC    ──────────→ │
  (인프라 구축)          │
     ↓                  │
 GR Education          │
  (역량 강화)   ──────→ ┘
```

- 진단이 문제를 발견하면 → 컨설팅이 분석하고 → IaC가 해결하고 → 교육이 재발 방지
- 모든 제품이 **같은 GR DB**와 **같은 GR Atlas**를 공유
- 고객은 하나의 제품에서 시작해서 자연스럽게 다른 제품으로 확장

## 4. 경쟁 우위와 해자 (Moat)

### 4.1 5대 경쟁 우위

| # | 우위 | 설명 | 복제 난이도 |
|---|------|------|------------|
| 1 | **지식 베이스** | 30,000 atom 사전 매핑 | 2년+ |
| 2 | **3D 좌표 체계** | 특허 가능한 고유 분류 시스템 | 높음 |
| 3 | **AI Ground Truth** | 할루시네이션 방지 검증 데이터셋 | 매우 높음 |
| 4 | **네트워크 효과** | 제품 매핑 ↑ → AI 정확도 ↑ → 사용자 ↑ | 자가 강화 |
| 5 | **데이터 플라이휠** | 사용할수록 분류 정확도 향상 | 시간 의존 |

### 4.2 vs 경쟁사

| 경쟁사 | 그들의 강점 | GR의 차별화 |
|--------|------------|------------|
| MITRE ATT&CK | 공격 기법 분류 | GR은 WHERE(인프라 컨텍스트) 추가 |
| Tenable/Qualys | 취약점 스캐닝 | GR은 좌표 기반 위험도 재산정 |
| Terraform | IaC 배포 | GR은 보안 정책 내장 IaC |
| NIST/ISO | 컴플라이언스 프레임워크 | GR은 좌표 기반 자동 매핑 |

## 5. 시장 진입 전략

### Phase 1: 커뮤니티 구축
- 프레임워크 사양 오픈 소스 공개
- GR DB 제한적 무료 API 제공
- 보안 커뮤니티에서 인지도 확보
- 초기 파트너 3-5개 확보

### Phase 2: B2B MVP
- 자동 진단 도구 유료 런칭
- Premium DB API 출시
- 첫 10개 유료 고객 확보
- 피드백 루프로 제품 개선

### Phase 3: 제품 확장
- GR IaC, Education 런칭
- 컨설팅 서비스 시작
- 파트너 에코시스템 구축
- "Powered by GR" 라이선싱

### Phase 4: 생태계 확장
- API 이코노미 (B2B2B)
- 자체 Fine-tuned LLM
- 글로벌 확장
- 산업별 특화 온톨로지
